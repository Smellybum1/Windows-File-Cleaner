# End-of-task Checklist

Use this before calling a Codex task complete.

## Code

- [ ] Implementation is complete.
- [ ] Changes are focused and relevant.
- [ ] Naming matches `docs/domain/glossary.md`.
- [ ] No forbidden synonyms were introduced.
- [ ] Existing conventions were followed.
- [ ] New production dependencies were approved or avoided.

## Tests

- [ ] Relevant tests were added or updated.
- [ ] Relevant test command was run.
- [ ] Lint/type/build checks were run where applicable.
- [ ] Manual checks were documented if automated tests were not possible.

## Docs

- [ ] `docs/domain/context.md` updated if domain concepts changed.
- [ ] `docs/domain/glossary.md` updated if terms changed.
- [ ] `docs/domain/context-map.md` updated if context boundaries changed.
- [ ] ADR added if the decision met the ADR threshold.
- [ ] Feature brief updated with completion notes.
- [ ] `.codex/progress.md` updated for meaningful packets.

## SkillOpt-inspired evidence gate

- [ ] Evidence from the task was recorded where useful.
- [ ] Any workflow/naming changes were bounded to the smallest useful doc edits.
- [ ] Rejected terms, designs, or assumptions were recorded if they may recur.
- [ ] No broad workflow rewrite was accepted without evidence from actual work.

## Final response

Include:

- What changed
- Files changed
- Tests run
- Docs updated
- ADRs added or skipped
- Open questions
- Follow-up work
- Any assumptions or risks
